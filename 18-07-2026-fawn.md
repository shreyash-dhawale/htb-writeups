## What I Learned
 
1. Anonymous FTP login is a common misconfiguration — anyone can access files without credentials
2. vsftpd version information revealed in banner — useful for CVE lookup
3. FTP transmits data in plaintext — credentials and files visible to network sniffers

## How I Would Detect This in Splunk
 
```spl
index=main sourcetype=syslog "anonymous"
| rex field=_raw "from (?P<src_ip>\d+\.\d+\.\d+\.\d+)"

```
 
---
 
## Remediation
 
- Disable anonymous FTP access unless explicitly required
- Replace FTP with SFTP or FTPS which encrypt traffic
- Restrict FTP access to specific IP ranges via firewall rules
