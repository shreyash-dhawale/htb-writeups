---
 
## Exploitation
 
**Observation:** Port 445 (SMB) is open. SMB shares sometimes allow null session (no credentials) access.
 
**Action:** Enumerated SMB shares using smbclient.
 
```bash
# List available SMB shares
smbclient -L <machine-ip> -N
 
# Connect to a share without credentials
smbclient \\\\<machine-ip>\\ShareName -N
```
 
**Result:** Found accessible share with no password required.

## What I Learned
 
1. SMB null sessions allow unauthenticated enumeration of shares — a common Windows misconfiguration
2. smbclient -L lists all available shares including hidden ones
3. SMB is one of the most commonly exploited services in real attacks (EternalBlue, WannaCry)

---
 
## Remediation
 
- Disable SMB null sessions via Group Policy
- Restrict SMB access to authorised IP ranges only
- Enable SMB signing to prevent relay attacks
- Upgrade to SMBv3 which has better security controls
